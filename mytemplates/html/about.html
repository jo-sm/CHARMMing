Source Code


The source code for CHARMMing can be found at source.charmming.org or at CHARMMing's Google Code site.

Using the site


CHARMMing has been tested with Mozilla Firefox versions 2+ (up to and including 21). The browser must be set to accept cookies. It should also work with Internet Explorer 7+ and recent versions of Safari, but these have not been tested as thoroughly. It should also work well with recent versions of Google Chrome, including the mobile versions for Android phones; it is the preferred browser for visualizing molecules with GLmol.


To report bugs, please e-mail btmiller@helix.nih.gov or RishiSingh@gmail.com, but please check the known problem list below to make sure it's not a duplicate of an issue we already know about.

Known Problems

  • Occasionally, the status box will not update when a job is submitted. If this happens, please restart the job.

  • Antechamber will combine all "bad" hetero-atoms into a single residue when it is used.

  • GENRTF also combines all "bad" hetero-atoms into a single residue when hydrogens are generated.

  • Generating hydrogens for "bad" hetero-atoms can cause problems with Antechamber.

  • Visualizing custom sequences does not work until a CHARMM operation is performed on them (since coordinates must be built).

  • Downloading screenshots taken with GLmol can cause severe glitches in Firefox. It is recommended to instead copy the image to the clipboard and paste it into an image editing program or similar for local storage.

Software


CHARMMing makes use of a variety of additional software.

  • qTip2: provides context-sensitive help.

  • Django: an open source Web application framework.

  • GreyBox: for various display functionality.

  • SD Menu: used to create the sidebar.

  • jqTableKit: for table sorting.

  • Open Babel: generates hydrogens for bad heteroatoms and generates SDF files for parameter fitting.

  • JSmol: for visualization of molecules and custom ligand building.

  • GLmol: for fast visualization of molecules on machines with limited resources, such as phones.

  • jQuery: for various display functionality as well as JSmol.

  • jQuery-UI: for error/information boxes.
